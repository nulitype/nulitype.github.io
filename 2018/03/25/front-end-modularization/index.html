模块化

以功能块为单位进行程序设计，实现其求解算法的方法称为模块化。原则是“高内聚，低耦合”，目的是为了降低程序复杂度，使程序设计，调试和维护等操作简单化。

为什么需要模块化

在 JavaScript 发展初期，代码只是简单地堆积在一起，随着项目代码越来越庞大，就会有很多问题，比如全局变量冲突，函数命名冲突，依赖关系处理等。

原始写法

模块是要实现某个功能，那么可以把实现功能的一组函数放在同一文件中。

函数 a1 和 a2 组成一个模块，其他文件先加载该模块，再对函数进行调用。这种做法的缺点是代码量庞大就很难保证不发生变量名冲突的问题，容易污染全局变量。

添加命名空间

为了解决上面写法的缺点，可以加入命名空间来管理模块，也就是使用单全局变量的模式。

把 a1 和 a2 两个函数都封装在对象 module_speical中，通过 module_speical.a1() 调用。
但是这种写法不能阻止私有变量被外部修改。

立即执行函数（IIFE）

立即执行函数能够形成一个独立的作用域，用立即执行函数作为一个“容器”，“容器”内部可以访问外部的变量，外部环境不能访问“容器”内的变量，所以在“容器”内定义的的变量不会与外部的变量发生冲突。

这种方式既避免了命名冲突，又使得私有变量不能被外部访问和修改。

CommonJS 规范

Node.js 应用由模块构成，采用 CommonJS 规范，通过全局方法 require 来加载模块。
特点：

  • 所有模块都有单独的作用域，不会污染全局变量
  • 重复加载模块只会加载第一次，后面都从缓冲读取
  • 模块加载的顺序是按照代码出现的顺序
  • 模块加载是同步的

AMD 规范

AMD 规范支持异步加载模块，规范中定义了一个全局的 define 函数。
define(id?, dependencies?, factory); // 模块标识 依赖模块的数组 初始化执行的函数或对象

RequireJs

RequireJS 是 AMD 规范的实现代表。实现了 JavaScript 文件的异步加载，避免网页失去响应，管理模块之间的依赖性，便于代码的编写和维护。

CMD 规范

CMD 规范中，一个模块就是一个文件。
define(factory), define 是一个全局函数，用来定义模块，接收 factory 参数，当 factory 是函数时，表示模块的构造方法，factory 执行时默认传入三个参数：require, exports, module。

define(function(require, exports, module)) {};

require 是一个方法，接收模块标识作为唯一参数，用来获取其他模块的接口。
exports 是一个对象，用来向外提供模块接口。除了使用 exports 对象增加成员外，也可以直接使用 return 向外提供接口。
